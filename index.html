「Wedコンテンツ作成」の成果

本テーマについて

このテーマでは、HTML,CSS,JavaScriptの基本を学び、これらを組み合わせてWebページを構成する方法を学ぶ。目標としては、Webページの構成、見た目、JSでのインタラクティブな処理についての基礎を完璧にする。

プロジェクト課題について

プロジェクト課題では、NHKの番組表、グルメ情報、世界の天気の検索の3つのテーマの中から好きなものを選び検索サイトを作る。 私は、グルメ情報を選びました。理由としては、3つの中で一番使ったことのあり身近なものだと思い選びました。グルメでは、htmlで作った検索キーに「居酒屋」「カフェ」といったお店のジャンルを書き、検索ボタンを押すとそれにあった店舗名、アクセス、予算などの情報がサイト画面に表紙されるというシステムです。 project.html

数当てゲーム

数当てゲームでは、1~10までの整数を3回のうちに当てるゲームです。ゲーム開始時にランダムで1つの正解を生成する。プレーヤーはテキストボックスに予想する数字を入力し、回答ボタンを押して判定をする。チャンスは3回までです。3回のうちに正解した場合は、「正解です。おめでとう！」と表示させる。3回のうちに正解できないと、「答えは、〇〇でした。すでにゲームは終わっています」と表示させる。 1回目と2回目で当たらない場合は、「まちがい、答えはもっと大きいですよ」「まちがい、答えはもっと小さいですよ」を表示させる。またクリア後に解答ボタンを押した場合も、答えは ◯ でした．すでにゲームは終わっています」と表示させる。 kazuate.html

各授業回の学習事項

第1回　WWW と HTML

今回の授業では、WWW（World Wide Web）仕組みと、Webコンテンツを作るHTMLの基礎を学習した。 Web表示の技術を理解した上で、ヘッダーやボディの構造、タグによる意味づけを理解した。さらに、リファレンスを活用したコーディング演習や、不具合を見つけるデバッグ手法まで学んだ。

演習の成果: 演習1-4:HTMLのさまざまな要素

第2回　CSS による Web ページの装飾

今回の授業では、CSSの基本構造や記述法、優先順位の仕組みを学び、Webページの見た目の装飾を習得した。 演習や開発者ツールを用いたデバッグを通じ、文字や余白の制御、効率的な外部参照の手法を実践的に理解した。

演習の成果: 演習2-3: browser-into.html

第3回　JavaScript の基本

今回の授業では、JavaScriptの基礎からWebページへの導入方法、Javaとの違い、基本構文を学習した。 また、エラーが出たときに自力で解決するためのデバッグ方法や、わからない時の調べ方を習得した。

演習の成果: 演習3-6: obj-practice.html

第4回　イベント処理とDOM(1)

イベント処理とDOMの仕組みを基礎から学び、input要素から入力されたものを取得する方法や、要素を検索してテキストと属性値を書き換える技術を学習した。 演習の足し算ページ作りでは、実践を通して実際のイベント処理やそのデバッグ方法まで修得した。

演習の成果: 演習4-10: tashizan.html

第5回　DOM(2)

本講義では、DOMの高度な機能と、要素の新規作成・編集・削除といった動的な操作方法について学習した。 idやclassの設定、ページの装飾などの知識を深め、演習や課題などの実践を通して、DOMの技術を修得した。

演習の成果: 演習5-7: dom2-exercise.html

工夫点、苦心点、強調点、感想

工夫した点は、プロジェクト課題のグルメ情報検索サイトにおいて、ユーザーが自由に「検索キー」を入力して結果を出せる仕組みを構築した点です。 inputタグを使ったあらかじめある選択肢からジャンルを選ばせるのではなく、ユーザ本人がテキストボックスに直接「居酒屋」「カフェ」といった単語を打ち込んで検索できるようにした。これによってより実際のサイトに近い形にすることができた。

最も苦労した点は、JavaScriptを用いた「数当てゲーム」の条件分岐と状態管理の実装です。 1から10までのランダムな正解を作る処理自体はスムーズにいきましたが、回答回数が「3回以内か否か」の判定と、「すでにゲームが終了しているか」の判定管理を組み合わせるロジックの構築に苦心しました。 特に、3回目の挑戦で間違えた際の「残念でした、答えは〇〇です」という終了メッセージの出力と、クリア後やゲームオーバー後にさらにボタンが押された場合に「すでにゲームは終わっています」と正しく制限をかける処理の優先するものが難しくて時間がとてもかかりました。 デベロッパーツールで変数 kaisu や進行状況フラグの挙動を追うことで、ようやく意図通りの完璧なゲームを完成させることができました。

強調点としては、本テーマを通じて、単にHTMLで文書の骨組みを作り、CSSで見た目を整えるだけでなく、JavaScriptを組み合わせることで「ユーザーの動きに応じてリアルタイムに表示が変わる動的なWebサイト」を自力で構築できるようになった点です。 非同期通信（Ajax）による画面をリロードしなくてもデータの取得できる技術や、画面上の要素を自由に作成・編集するDOM操作の仕組みを修得できたことです。

最近は他の授業でもHTML、CSS、JavaScriptを扱うことが増えていましたが、この授業ではより深い内部の仕組みや、実践的なデバッグ方法まで徹底的に学ぶことができました。実は去年まではHTMLの使い方が全くわからず苦手意識がありましたが、今回の講義と演習を経て構造を正しく理解し、今では自信を持って記述できるようになりました。 特にCSSは、自分の思い描いた通りのデザインやレイアウトに画面を装飾できるのがとてもに楽しくて、モチベーションに繋がりました。一方で、JavaScriptに関しては正直プログラミング特有の論理的思考やDOMの高度な機能など、まだ難しいと感じる部分も多かったです。しかし、エラーの原因を自力で見つけて解決するデバッグの楽しさも知ることができたため、今後もさらに理解を深めていきたいです。
